1

Dashboard — Platform-wide health check

What you see

  • 4 KPI cards — Total Investors, Active Funds, Team Members, Aggregate Committed Capital
  • Fund health table — fund name, manager, investor count, % to close
  • Numbers update in real-time as investors move through the pipeline

Why it matters

  • Instant visibility into fundraising progress across the entire platform
  • Quickly spot stalled funds or pipeline bottlenecks
  • Validate whether team activity is translating into committed capital
2

Users — Manage fund manager accounts

What you see

  • Table: name, email, role, status (Active / Inactive), last login timestamp

What you can do

  • Create User — name + email → temp password shown once → user sets password on first login
  • Deactivate — toggle Active off to block login without deleting data
  • Reset password — generate a new temp password at any time
3

Funds — Create and configure SPVs and funds

Each fund is a fully isolated environment.

What you see

  • Card grid: name, type, currency, target raise, assigned managers

What you can do

  • Create Fund — name, type (SPV / Fund), currency, target raise, close date, thesis, sectors, regions
  • Edit Fund — update any field; changes apply immediately
  • Auto pipeline — creating a fund provisions all 13 pipeline stages automatically
4

Assignments — Map fund managers to funds

Controls exactly what each user can see.

  • Grid of funds (rows) × managers (columns) — check a cell to grant access
  • One manager can be assigned to multiple funds
  • Data is fully isolated — managers only see assigned funds
  • Changes take effect immediately, no logout required
5

All Investors — Unified view across all funds

What you see

  • Columns: name, fund, investor type, nationality, stage, relationship strength, expected ticket
  • Relationship strength is colour-coded: cold / warm / direct

What you can do

  • Search — name, firm, email
  • Filter — fund, stage, nationality, investor type, relationship strength
  • Open profile — click any row for a read-only profile drawer
6

Duplicate Detection — AI-powered cross-fund deduplication

  • Fuzzy matching on three signals: (1) investor name, (2) email addresses, (3) LinkedIn URLs
  • Suspected duplicates appear as side-by-side pairs with a confidence score (0–100%)
  • Merge — choose a winner; all tasks, call logs, pipeline history, and interactions are preserved
  • Dismiss — mark as not a duplicate so it stops appearing in the queue
7

Investor Requests — Queue from Chrome Extension captures

What you see

  • Card: capturing member, source URL, capture date, extracted fields
  • Status badge: Pending / Approved / Rejected

What you can do

  • Approve — adds investor to the selected fund instantly
  • Reject — discards capture (optional reason), retained in archive
  • Filter — All / Pending / Approved / Rejected
8

Personas Analytics — Platform-wide persona coverage

  • KPI cards — total personas, investors with 50%+ match score, average match score platform-wide
  • Per-fund table — persona count and % of investors matched at 50%+
  • Persona leaderboard — all personas ranked by matched investor count
  • Unmatched pool — investors below 50% on all personas (signals for new personas)
9

Feedback Responses — User testing feedback

What you see

  • Stats bar — total responses, avg intuitiveness (1–5), % requesting more AI
  • Response table — submitter, role, date, score, comment preview; expand row for full response

What you can do

  • Filter — by role (Admin / Fund Manager), date range, keyword search
  • Export CSV — download all responses as a spreadsheet
