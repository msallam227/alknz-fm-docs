1. Dashboard

Platform-wide health check

The landing screen after admin login. A real-time platform-wide overview.

What you see

  • 4 KPI cards: Total Investors, Active Funds, Team Members, Aggregate Committed Capital
  • Fund health table: fund name, assigned manager, investor count, % to close
  • Real-time updates as investors move through the pipeline

Why it matters

  • Instant visibility into fundraising progress across the entire platform
  • Quickly spot stalled funds or pipeline bottlenecks
  • Validate whether team activity is translating into committed capital
2. Users

Manage fund manager accounts

What you see

  • Table: name, email, role, status (Active/Inactive), last login timestamp

What you can do

  • Create User: name + email → temp password shown once → user sets password on first login
  • Deactivate: toggle Active off to block login without deleting data
  • Reset password: generate a new temp password any time
3. Funds

Create and configure SPVs and funds

Each fund is a fully isolated environment.

What you see

  • Card grid: name, type, currency, target raise, assigned managers

What you can do

  • Create Fund: name, type (SPV/Fund), currency, target raise, close date, thesis, sectors, regions
  • Edit Fund: update any field; changes apply immediately
  • Auto pipeline: creating a fund provisions all 13 pipeline stages automatically
4. Assignments

Map fund managers to funds

Controls exactly what each user can see.

How it works

  • Grid of funds (rows) × managers (columns)
  • Check a cell to grant access; one manager can be assigned to multiple funds
  • Data is isolated: managers only see assigned funds
  • Changes apply immediately (no logout required)
5. All Investors

Unified investor view across all funds

What you see

  • Columns: name, fund, investor type, nationality, stage, relationship strength, expected ticket
  • Relationship strength is colour-coded: cold / warm / direct

What you can do

  • Search: name, firm, email
  • Filter: fund, stage, nationality, investor type, relationship strength
  • Open profile: click any row for read-only drawer
6. Duplicate Detection

Finds investors across multiple funds

How it works

  • Fuzzy matching across: (1) name, (2) emails, (3) LinkedIn URLs
  • Shows suspected pairs with confidence score (0–100%)
  • Merge: pick a “winner”; preserves tasks, call logs, pipeline history, and interactions
  • Dismiss: mark as not a true duplicate to remove from queue
7. Investor Requests

Queue from Chrome Extension captures

What you see

  • Card: capturing member, source URL, capture date, extracted fields
  • Status: Pending / Approved / Rejected

What you can do

  • Approve: adds investor to selected fund instantly
  • Reject: discard (optional reason), retained in archive for audit
  • Filter: All / Pending / Approved / Rejected
8. Personas Analytics

Platform-wide persona coverage

What you see

  • KPI cards: total personas, investors with 50%+ match, average match score
  • Per-fund table: persona count and % matched at 50%+
  • Persona leaderboard ranked by matched investor count
  • Unmatched pool: investors below 50% on all personas (signals for new personas)
9. Feedback Responses

User testing feedback (admins + fund managers)

What you see

  • Stats: response count, avg intuitiveness (1–5), % requesting more AI
  • Table: submitter, role, date, score, comment preview; expand for full response

What you can do

  • Filter: role, date range, keyword search
  • Export CSV: download all responses
