Tab 1 — Capital Overview

Live fund metrics

The first screen after Fund Manager login. Updates immediately as stages/ticket sizes change.

What you see

  • 4 KPI cards: Target Raise, Total Committed, Total Funded, % to Close
  • Ticket size distribution chart (capital by size bucket)
  • Stage funnel (investor count per pipeline stage)

How to use it

  • Use funnel to find bottlenecks (where investors stall)
  • Use distribution to see if raise is reliant on whales vs broad base
  • Keep ticket sizes updated; everything aggregates from that number
Tab 2 — Task Manager

All tasks in one list

The bell icon shows a live count of open tasks.

What you see

  • Investor name, task description, type (⚡ Auto / Manual)
  • Priority (high/medium/low), due date, completion checkbox

Task types

  • ⚡ Auto: generated when investor moves stages (standard checklist)
  • Manual: created via +, link to investor, set due date/priority/notes
  • Filter by investor, stage, priority, completion
Tab 3 — Investor Profiles

Fund investor database

List columns

  • Name, investor type, nationality, pipeline stage
  • Relationship strength badge, expected ticket size
  • Persona match % badge

Investor detail panel (4 tabs)

TabWhat’s inside
Identity Full name, type, nationality, age, job title, firm, LinkedIn, wealth level
Context Relationship strength, decision role, preferences (sectors, check size, geography), notes
Contact Primary email + up to 4 secondary emails, phone, WhatsApp, point of contact
Personas AI match score progress bars for each persona; updates after analysis runs

Edit mode: Click Edit → modify fields → Save to apply immediately.

Tab 5 — Communication Center

Inbox + Call Logs + Templates

Inbox

  • Connect Gmail via OAuth (Settings → Gmail → Connect)
  • Investor emails auto-tagged by matched addresses
  • Compose/reply inside the portal

Call Logs

  • Log calls: investor, date, outcome, notes, next steps
  • Optional: auto-create follow-up task linked to the investor

Templates

  • Email templates with merge vars:
  • {{investor_name}}, {{fund_name}}, {{ticket_size}}
  • Select template when composing → auto-populates
Tab 6 — Research Capture

Review Chrome Extension captures

Accept flow

  1. Select a fund from dropdown on the capture card
  2. Click Accept
  3. Investor is created immediately
  4. Portal switches to Investor Profiles and opens the new profile automatically

Reject

Reject discards the capture with an optional reason. Rejected stays archived with status.

Tab 7 — Personas

Define archetypes + score investors (0–100%)

What you can do

  • Create Persona: type, nationalities, sectors, age range, ticket size range
  • Run AI Analysis: scores every investor vs every persona (0–100%)
  • Analyze & Suggest: AI suggests archetypes not covered by existing personas

Where scores appear

  • Investor list: badge %
  • Investor detail: Personas tab progress bars
  • Use 50%+ as “match” threshold for coverage tracking
